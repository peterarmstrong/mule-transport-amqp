h1. Overview
Connects to AMQP brokers

h1. Classpath Settings
The latest Mule Amqp Transport has been tested with Mule [3.1.0-SNAPSHOT|http://www.mulesoft.org/display/MULE/Download].

h2. Mule Standalone
To set up the Mule classpath to use the Amqp Transport, you must first [download|http://dist.muleforge.org/amqp/] the Mule Amqp distribution. Then simply take all the JARs inside the distribution {{lib/}} directory and put them in {{MULE_HOME/lib/user}}.

h2. Maven
If you are using Maven, you can use the MuleForge repositories and add a dependency to Jersey that way. First add the MuleForge repository to your POM:
{code:xml}
<repositories>
    <repository>
      <id>muleforge</id>
      <name>MuleForge Repository</name>
      <url>http://repository.muleforge.org</url>
    </repository>
</repositories>
{code}

Then add the dependency:
{code:xml}
<dependencies>
  <dependency>
    <groupId>org.mule.transports</groupId>
    <artifactId>mule-transport-amqp</artifactId>
    <version>1.0-SNAPSHOT</version>
  </dependency>
</dependencies>
{code}

h1. Using the Amqp Transport

Once you have included the Amqp Transport in your project or in the Mule distribution you can use it by adding the amqp namespace in your Mule XML configuration file -

{code:xml}
<?xml version="1.0" encoding="UTF-8"?>
<mule xmlns="http://www.mulesoft.org/schema/mule/core/3.1"
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
      xmlns:amqp="http://www.mulesoft.org/schema/mule/amqp/3.1"
      xsi:schemaLocation="
       http://www.mulesoft.org/schema/mule/core/3.1
http://www.mulesoft.org/schema/mule/core/3.1/mule.xsd
       http://www.mulesoft.org/schema/mule/amqp/3.1
http://www.mulesoft.org/schema/mule/amqp/3.1/mule-amqp.xsd">

</mule>
{code}

Now you can start using Amqp elements in your configuration.
